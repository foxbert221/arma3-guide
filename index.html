武装突袭3 脚本开发从零到入门面向零编程基础玩家的 SQF 脚本学习手册

前言：为什么学 Arma 3 脚本？

脚本能做什么？

Arma 3 内置了一套强大的脚本引擎，学会写脚本后，你可以——

  • 自制任务：设计属于自己的剧情任务，控制 AI 行为、触发事件、管理任务目标
  • 修改游戏玩法：自定义复活系统、装备发放、载具刷新等核心机制
  • 创建动态战场：随机生成巡逻队、动态天气、随时间变化的战场局势
  • 开发模组：从简单的脚本模组到完整的游戏改造 Mod

学脚本需要什么基础？

只要你会用电脑、玩过 Arma 3，零代码基础完全 OK！本手册会从最基础的概念讲起，不预设任何编程经验。

你只需要：一台能跑 Arma 3 的电脑、Steam 上的 Arma 3 本体，以及一颗愿意尝试的心。

第一章 · 准备工作（环境搭建）

1 Arma 3 Tools 安装

Arma 3 Tools 是官方提供的开发工具包，包含 Addon Builder（打包工具）等必备组件：

  1. 打开 Steam 客户端
  2. 点击顶部菜单「库」→ 将筛选器从"游戏"切换为「工具」
  3. 在列表中找到 Arma 3 Tools，右键 → 安装
  4. 等待下载完成即可
如果你只写脚本、不做模组发布，前期可以不装 Arma 3 Tools。但后续打包 PBO 时一定会用到，建议提前装好。

2 推荐的代码编辑器

强烈推荐 Visual Studio Code（免费、轻量、插件丰富）：

  1. 前往 code.visualstudio.com 下载安装
  2. 打开 VS Code，点击左侧扩展图标（或按 Ctrl+Shift+X）
  3. 搜索并安装以下插件：SQF Language（语法高亮 + 自动补全）

装好插件后，你写的 .sqf 文件就会有语法高亮和智能提示，大大降低手滑写错的概率。

其他可选编辑器：Notepad++（装 SQF 高亮插件）、Sublime Text。但 VS Code 是目前社区的主流选择。

3 3DEN 编辑器基础

3DEN 是 Arma 3 内置的任务编辑器，你可以用它摆放单位、设置触发器、关联脚本。

打开方式：游戏主菜单 → 「编辑器」→ 选择地图 → 新建。

基本操作：

  • 双击地图：放置单位/物体
  • 右键 → 触发器（Trigger）：创建触发区域
  • 右键 → 标记点（Marker）：放置地图标记
  • 选中物体后，在右侧面板可以设置名称（Variable Name）、初始化代码（Init）等
在编辑器中给单位/触发器起的 Variable Name，就是在脚本中引用它的变量名。一定要起一个有意义、好记的名字！

4 认识文件结构

一个典型的 Arma 3 任务文件夹（保存在 我的文档\Arma 3\missions\ 或 MPMissions\）包含以下核心文件：

文件名作用必需？
mission.sqm任务主文件，存储地图上所有物体、触发器、标记的位置和属性（3DEN 编辑器自动生成）✅ 是
init.sqf任务初始化脚本，任务开始时会自动执行（脚本开发的"入口文件"）❌ 建议有
initPlayerLocal.sqf每个玩家进入游戏时执行一次（多人联机常用）❌ 多人推荐
description.ext任务配置文件：复活设置、重生装备、天气、加载画面等❌ 建议有
onPlayerRespawn.sqf玩家复活时执行（需要 description.ext 中开启复活）❌ 按需
最关键的文件是 init.sqf——它就像"任务的入口"，大部分脚本逻辑都从这里开始。

5 PBO 文件是什么？

PBO（Packed Bohemia Object）是 Arma 系列的标准打包格式，类似 ZIP 压缩包。任务文件夹打包成 PBO 后才能发布到创意工坊或在多人服务器上使用。

打包方式：

  • Arma 3 Tools → Addon Builder：右键任务文件夹 → 打包
  • 第三方工具：PBO Manager（更简单直观，社区推荐）
学习阶段不需要打包！直接在编辑器中「预览」（Play）就能测试脚本。只有发布时才需要 PBO。

第二章 · SQF 脚本语言入门

SQF 是什么？

SQF（Status Quo Function）是 Arma 系列的脚本语言，语法类似 C 语言家族。你写的每一个脚本文件都以 .sqf 为后缀。

第一个脚本：Hello Arma！

创建一个文件 init.sqf，写入以下内容：

// 这是我的第一个脚本！
systemChat "Hello Arma! 脚本世界我来了！";
hint "欢迎来到武装突袭3脚本开发！";

把这个文件放到任务文件夹里，进入游戏就能看到左下角聊天区出现提示文字，屏幕中央弹出 Hint 提示框。

注释怎么写？

注释是写给人类看的说明文字，游戏引擎会忽略它们。养成写注释的好习惯！

// 这是单行注释，从双斜杠到行尾都是注释

/*
   这是多行注释
   可以写很多行
   适合写大段说明
*/

变量：存储数据的容器

变量就像一个带标签的盒子，你可以把数据放进去、随时取出来用。

局部变量（下划线开头）

private _myNumber = 42;              // 只在当前脚本块内有效
private _playerName = name player;    // 获取玩家名字

全局变量（无下划线，整个任务都能访问）

missionDone = false;                    // 所有脚本都能读写
enemyCount = 12;

特殊变量 _this

_this 是 SQF 中最重要的特殊变量。当脚本被调用时自动传入，代表"调用者传给脚本的东西"。

// 在单位的 Init 框中写：
// [player, 100] execVM "giveMoney.sqf";

// giveMoney.sqf 中：
params ["_targetUnit", "_amount"];
systemChat format ["给 %1 发了 %2 块钱", name _targetUnit, _amount];

数据类型

类型说明示例
数字 (Number)整数或小数42、3.14、-5
字符串 (String)文本，用双引号包起来"Hello"、"张三"
布尔 (Boolean)真或假true、false
数组 (Array)一组数据，用方括号[1, 2, 3]、["红", "蓝"]
代码块 (Code)一段待执行的代码，用花括号{ hint "你好"; }
对象 (Object)游戏中的单位、载具等player、this
空值 (nil)啥也没有nil、objNull

运算符

类别符号含义示例
算术+加3 + 4 → 7
-减10 - 3 → 7
*乘5 * 6 → 30
/除20 / 4 → 5
%取余（模）10 % 3 → 1
比较==等于5 == 5 → true
!=不等于5 != 3 → true
>大于8 > 3 → true
<小于2 < 10 → true
>=大于等于
<=小于等于
逻辑&&并且（AND）两个条件都满足
||或者（OR）任一条件满足
!取反（NOT）!true → false

条件判断 if...then...else

private _hp = damage player;

if (_hp > 0.5) then {
    hint "你受了重伤！快找掩体！";
} else {
    hint "状态良好，继续前进！";
};
判断相等用 ==（两个等号），不是 =（一个等号）。这是新手最常见的错误！

循环

for 循环（知道要循环多少次）

// 在玩家周围生成5个靶子
for "_i" from 1 to 5 do {
    private _pos = player getPos [random 30, random 360];
    "TargetP_Civ_F" createVehicle _pos;
    systemChat format ["已生成第 %1 个靶子", _i];
};

while 循环（条件满足就一直循环）

// 只要玩家还活着，每10秒报告一次位置
while {alive player} do {
    systemChat format ["当前位置：%1", getPos player];
    sleep 10;
};

forEach 循环（遍历数组中的每个元素）

private _fruits = ["苹果", "香蕉", "橘子"];
{
    systemChat format ["我喜欢吃 %1", _x];
} forEach _fruits;
在 forEach 循环中，_x 代表当前遍历到的元素，_forEachIndex 代表当前索引（第几个）。

函数定义与调用

把一段可以复用的代码打包成函数：

// 定义一个函数：在指定位置生成烟雾
my_fnc_createSmoke = {
    params ["_pos", "_color"];
    
    private _smoke = "SmokeShell" createVehicle _pos;
    systemChat format ["在 %1 生成了一团 %2 烟雾", _pos, _color];
};

// 调用函数
[getPos player, "红色"] call my_fnc_createSmoke;

第四章 · 实战案例（从零到上手）

以下所有案例都可以直接复制到你的任务中运行。建议先通读一遍代码，然后逐个亲手敲一遍。

案例1：进游戏自动发提示消息

难度：⭐   学习重点：init.sqf 入口 + systemChat + hint

创建 init.sqf 文件：

// init.sqf —— 任务开始时自动执行

systemChat "========== 欢迎来到自定义任务 ==========";
sleep 2;  // 等2秒，让玩家先适应一下

hint format [
    "你好，%1！\n任务目标：消灭所有敌人。\n祝你好运！",
    name player
];

sleep 5;
systemChat "提示：打开地图 (M) 查看任务标记。";

案例2：玩家走到区域触发剧情对话

难度：⭐⭐   学习重点：waitUntil + 触发器替代 + 字幕对话

// 剧情触发区域（坐标来自 3DEN 编辑器中的触发器位置）
private _triggerPos = getMarkerPos "trigger_area";
private _triggerRadius = 20;

// 等待玩家进入区域
systemChat "前往地图上的标记点...";
waitUntil {
    player distance _triggerPos < _triggerRadius
};

// 触发！播放剧情对话
titleText ["无线电响起...", "BLACK IN", 2];
sleep 2;

systemChat "指挥部：士兵，你已到达目标区域。";
sleep 3;
systemChat "指挥部：搜索该区域，找到情报文件。";
sleep 3;
systemChat "指挥部：完毕。";

// 创建任务
private _task = player createSimpleTask ["findIntel"];
_task setSimpleTaskDescription ["在区域内搜索情报文件", "搜寻情报", ""];
_task setSimpleTaskDestination _triggerPos;

案例3：杀死指定敌人完成任务

难度：⭐⭐   学习重点：addEventHandler + 条件监测 + 任务系统

在编辑器中给目标敌人起名 targetOfficer，然后在 init.sqf 中：

// 创建消灭任务
private _killTask = player createSimpleTask ["killOfficer"];
_killTask setSimpleTaskDescription [
    "消灭敌军指挥官 targetOfficer",
    "刺杀指挥官",
    "目标已标记"
];
_killTask setSimpleTaskDestination getPos targetOfficer;

// 监听目标死亡事件
targetOfficer addEventHandler ["Killed", {
    params ["_unit"];
    
    _killTask setTaskState "SUCCEEDED";
    ["TaskSucceeded", ["指挥官已被消灭！"]] call BIS_fnc_showNotification;
    systemChat "✅ 任务完成：敌军指挥官已被击毙。";
}];

案例4：创建随机巡逻队

难度：⭐⭐⭐   学习重点：循环 + 随机位置 + 创建单位 + 路径点

// 生成随机巡逻队函数
my_fnc_spawnPatrol = {
    params ["_center", "_radius", "_count"];
    
    private _group = createGroup east;
    
    // 创建巡逻队成员
    for "_i" from 1 to _count do {
        private _pos = _center getPos [random _radius, random 360];
        _group createUnit ["O_Soldier_F", _pos, [], 0, "NONE"];
    };
    
    // 添加巡逻路径点（随机走几个点）
    for "_i" from 1 to 3 do {
        private _wpPos = _center getPos [random _radius, random 360];
        private _wp = _group addWaypoint [_wpPos, 0];
        _wp setWaypointType "MOVE";
        _wp setWaypointBehaviour "SAFE";
        _wp setWaypointSpeed "LIMITED";
    };
    
    // 最后一个路点设为循环
    private _lastWP = _group addWaypoint [_center, 0];
    _lastWP setWaypointType "CYCLE";
    
    systemChat format ["一支 %1 人巡逻队已出动", _count];
};

// 生成3支巡逻队
[getPos player, 200, 4] call my_fnc_spawnPatrol;
[getPos player, 200, 3] call my_fnc_spawnPatrol;
[getPos player, 200, 5] call my_fnc_spawnPatrol;

案例5：复活系统（Respawn with Loadout）

难度：⭐⭐⭐   学习重点：description.ext + onPlayerRespawn + 装备保存

第一步：创建 description.ext：

// description.ext
respawn = "BASE";                  // 基地复活模式
respawnDelay = 5;                 // 复活等待时间（秒）
respawnOnStart = 1;               // 游戏开始就标记复活点（-1=关）

第二步：创建 onPlayerRespawn.sqf（保存装备并在复活后恢复）：

// onPlayerRespawn.sqf —— 玩家复活时自动执行
params ["_unit", "_corpse"];

// 从尸体上找回装备
private _loadout = getUnitLoadout _corpse;
_unit setUnitLoadout _loadout;

systemChat "你已经复活，装备已恢复。";
hint "重新投入战斗！";

案例6：动态天气与时间系统

难度：⭐⭐   学习重点：天气命令 + 时间推移 + 循环脚本

// 在 init.sqf 中启动
[] spawn {
    // 随机初始天气
    private _overcast = random 1;         // 云量 0(晴)-1(阴)
    private _fog = random 0.3;            // 雾量
    private _rain = random 0.5;           // 雨量
    
    300 setOvercast _overcast;          // 300秒内过渡到设定云量
    60 setFog _fog;                      // 雾过渡
    60 setRain _rain;                    // 雨过渡
    
    // 时间设为黄昏（18:00）
    skipTime (18 - daytime);
    
    systemChat format [
        "天气：云量%1 雾量%2 雨量%3 | 时间：18:00",
        _overcast, _fog, _rain
    ];
    
    // 循环：每5分钟随机微调天气（模拟动态变化）
    while {true} do {
        sleep 300;  // 5分钟 = 300秒
        private _change = (random 0.2) - 0.1;  // -0.1 到 +0.1 随机变化
        _overcast = (_overcast + _change) max 0 min 1;  // 限制在 0-1 之间
        120 setOvercast _overcast;
        systemChat "天气在变化...";
    };
};

第五章 · 调试与排错

写代码一定会出错。能快速定位和修复错误，才是真正会写脚本的标志。本章教你必备的调试技能。

调试工具

方法用法适用场景
systemChat在聊天区输出变量值快速检查变量对不对
diag_log输出到日志文件正式调试，不干扰游戏画面
hint str弹出提示框显示变量需要立即看到结果
Debug Console游戏内按 Esc → Debug Console实时执行代码、查看变量
// 调试示例：检查变量值
private _myVar = 42;

systemChat format ["[DEBUG] _myVar = %1", _myVar];
diag_log format ["[DEBUG] player position = %1", getPos player];
hint format ["[DEBUG] enemy alive: %1", alive targetOfficer];
日志文件位置：%localappdata%\Arma 3\*.rpt — 这里记录了游戏运行期间的所有输出。

常见报错解读

报错信息含义常见原因
Undefined variable变量未定义拼写错误、忘记声明、作用域问题
Type String, expected Number类型不匹配把字符串当数字用了（如 "5" + 3）
Generic error in expression表达式语法错误括号不匹配、少分号、命令拼错
Error Zero Divisor除以零除数变量为 0
Missing ;缺少分号语句末尾忘了加分号 ;

启动参数

在 Steam 中右键 Arma 3 → 「属性」→ 「启动选项」，添加以下参数：

参数作用
-showScriptErrors在屏幕上显示脚本错误（强烈推荐！）
-debug开启调试模式
-noLogs关闭日志（不要加这个）
-filePatching允许直接加载未打包文件（开发必备）
推荐启动参数：-showScriptErrors -filePatching，开发过程中始终开着。

多人联机调试注意事项

  • 客户端脚本（如 initPlayerLocal.sqf）只在玩家自己的电脑上运行
  • 服务器脚本在服务器上运行，所有玩家都能看到效果
  • 使用 isServer 判断当前是否在服务器上执行
  • 使用 remoteExec 在服务器上远程执行命令

第六章 · 进阶学习路径

Event Handlers（事件处理器）

Event Handler 是让脚本"监听"游戏事件的机制——当某件事发生时自动触发你的代码。

// 监听玩家被击中
player addEventHandler ["Hit", {
    params ["_unit", "_source", "_damage"];
    systemChat format ["你受到了 %1 点伤害！", _damage];
}];

// 常用事件：Killed / Hit / Fired / GetIn / GetOut / Respawn / HandleDamage

多人联机脚本要点

在多人游戏中，脚本运行位置至关重要。核心概念：

  • Locality（本地性）：每个对象只存在于一台机器上。玩家在自己电脑上、AI 在服务器上
  • remoteExec：在指定机器上远程执行代码
  • publicVariable：将变量同步到所有客户端
  • initPlayerLocal.sqf vs initServer.sqf：分别在本机和服务器执行
多人脚本是 Arma 脚本开发中最复杂的部分。建议先把单机脚本练熟，再逐步接触多人联机内容。

编组与航点

  • 编组：选中多个单位 → F2 进入编组模式 → 拖线连接 → 单位共享一个组长（Leader），AI 自动跟随
  • 航点（Waypoint）：按 F4 切换到航点模式 → 选中单位 → 右键地图添加航点
  • 航点类型：Move / Sentry / Guard / Hold / Cycle / Destroy / Seek and Destroy / Get In / Load / Unload / Land / Scripted 等
  • 航点属性：Timeout（超时等待）、Condition（条件表达式）、On Act（激活脚本）、Completion Radius（完成半径）
Waypoint 的 Cycle 类型可实现巡逻循环：将最后一个航点设为 Cycle 并指向第一个航点，AI 会无限循环巡逻。
Variable Name 只能包含字母、数字和下划线，且不能以数字开头。命名建议：myUnit、enemy_1、objTarget，避免 unit、leader 等可能与引擎保留字冲突的名称。

触发器系统详解

触发器（Trigger）是编辑器中实现事件驱动的核心机制，相当于"当某个条件满足时，执行某段代码"。

创建与基本设置

  • 创建：按 F3 切换到触发器模式 → 双击地图放置
  • 或：右键地图 → 新建触发器

触发器属性一览

属性说明示例
Activation（激活方）哪些阵营/组/单位可触发BLUFOR / OPFOR / Any / Civilian
Activation Type进入 / 离开 / 驻留Present / Not Present / West Detected
Repeatable是否可重复触发勾选后每次满足条件都会执行
Server Only仅在服务端触发（多人优化）勾选避免每台客户端都触发
Condition自定义激活条件表达式this && alive unit1
On Activation激活时执行的代码"任务完成！" remoteExec ["hint"]
On Deactivation条件不再满足时执行的代码hint "敌人离开了区域"
Timeout条件持续满足多久才触发（防抖）Min/Mid/Max 秒数
Countdown条件满足后延迟多久触发Min/Mid/Max 秒数

触发器区域设置

  • 形状：椭圆（Ellipse）或 矩形（Rectangle）
  • 尺寸：a（长轴/宽度）、b（短轴/高度），单位为米
  • 角度：旋转角度
  • 选中触发器后，地图上显示虚线边界，可拖动角点调整大小

Condition 字段详解

默认值为 this，表示"激活方进入区域即为真"。可自定义组合条件：

// 默认：激活方进入即触发
this

// 激活方进入 且 unit1 仍活着
this && alive unit1

// 激活方进入 且 任务状态为"进行中"
this && taskState task1 == "ASSIGNED"

// 区域内 BLUFOR 人数大于 3
count (thisList select {side _x == west}) > 3

// 特定变量为真时触发
myFlag

常用触发器模式

① 区域进入检测：

// Activation: BLUFOR, Present
// On Act:
"友军已进入目标区域" remoteExec ["hint"];
task1 setTaskState "SUCCEEDED";

② 任务完成检测：

// Condition:
! alive enemyCommander
// On Act:
"指挥官已被击杀！" remoteExec ["hint"];
task_kill setTaskState "SUCCEEDED";

③ 定时触发：

// Condition:
time > 300  // 任务开始5分钟后触发
// On Act:
"增援部队已到达！" remoteExec ["hint"];
["B_Soldier_F", getPos spawnPoint, [], 5] call BIS_fnc_spawnGroup;
thisList 是触发器内置变量，存储当前在区域内的所有实体数组。可以用 count thisList 获取区域内实体数量，或 thisList select {side _x == east} 筛选特定阵营。

模块同步连接

部分模块需要通过同步（Sync）连线关联目标：

  • 按 F5 进入同步模式
  • 从模块拖线到目标实体（单位/载具/触发器等）
  • 连线成功后，模块将对同步目标生效
Spawn AI 模块是做大规模战斗场景的利器。可以设置生成数量、阵营、行为模式、动态增强（随时间增加敌人数量），无需手写一个 spawn 脚本。

标记是地图上的视觉指示器，玩家可在地图（M键）上看到。

附录 C：常见问题 FAQ

Q：init.sqf 不执行怎么办？
A：检查文件名是否拼写正确（大小写敏感）、文件编码是否为 UTF-8 without BOM、文件是否放在任务文件夹根目录。

Q：为什么脚本里的中文显示乱码？
A：确保 .sqf 文件保存为 UTF-8 without BOM 编码。在 VS Code 右下角可以查看和修改编码。

Q：如何在游戏中实时测试单行代码？
A：按 Esc → 点击左下角 "Debug Console" → 在输入框中输入代码 → 点击 "LOCAL EXEC" 或 "SERVER EXEC"。

Q：sleep 命令为什么不生效？
A：sleep 只能在 spawn 或 execVM 创建的子线程中使用，不能直接在 init.sqf 的主线中用（除非 init.sqf 本身是用 spawn 调用的）。

Q：脚本中的 == 和 = 有什么区别？
A：== 是比较（判断是否相等），= 是赋值（把右边的值存到左边变量）。这个区别一定要记牢！

Q：如何知道一个命令的具体用法？
A：最快的方法：Google 搜索 arma 3 命令名，第一条结果通常是 BI Wiki 的文档页面，有完整参数说明和代码示例。
